# Submit your issues for clirnet-frontend


# Create the Bug Report Document

- Open Microsoft Word (desktop, web, or mobile app) and create a new document in your OneDrive (e.g., via OneDrive.com or the Word app).
- Use the updated bug report template (see below) to structure the content. Include all relevant details, such as:
  - Description of the issue.
  - Steps to reproduce.
  - Expected vs. actual behavior.
  - Screenshots, logs, or attachments embedded directly in the Word file.
- Save the file with a descriptive name, e.g., `Bug-Report-Login-Failure-v1.docx`.

# Share the Document Organization-Wide

- In the Word app or OneDrive.com, click Share (top-right corner).
- Set permissions to Anyone with the link can edit (or view if read-only is preferred initially).
- Under "Specific people," select People in [your organization] (clirnet.com) to restrict access to org members only. This ensures automatic access for all team members without individual invites.
- Copy the sharing link (it will look like `https://1drv.ms/w/...`).

# Create the GitHub Issue

- Go to your issues-only repository on GitHub.
- Click New issue and select the Bug report template (updated as below).
- Fill in:
  - **Title:** A concise summary (e.g., "Login fails on mobile Safari after password reset").
  - **Body:** A brief overview (1-2 paragraphs) of the bug, including environment details (OS, browser, version).
  - **OneDrive Link:** Paste the full sharing link in a dedicated section.
